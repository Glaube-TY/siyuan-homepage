# 项目全局第一性原则

本项目的所有功能都必须以"不破坏用户原有环境"为第一前提。插件可以增强思源的使用体验，但不能改变思源本体、用户主题、其他插件或用户文档的既有行为；任何功能、样式、数据读写和自动化操作，都必须限定在明确的插件作用域内，并且尽量可解释、可回退、可关闭。

## 样式隔离原则

插件样式只能影响插件自己的界面，不能污染思源全局样式、默认主题样式、用户自定义主题或其他插件界面。

使用思源内置组件类和主题变量时，必须先确认它们是真实存在且稳定的思源样式能力。可以复用思源的基础类名和 CSS 变量来保持视觉一致，但不能在插件中全局覆盖 `.b3-button`、`.b3-text-field`、`.b3-switch` 等思源通用组件类，也不能定义过于通用的全局类名影响外部页面。

所有自定义样式都应绑定在插件根容器、功能模块容器或组件局部类名下。必须避免无作用域的全局选择器，避免直接重写 `body`、`:root`、`.layout`、`.b3-*` 等全局对象。确实需要全局样式时，必须说明原因，并尽量使用插件专属前缀和最小影响范围。

当样式需要适配主题时，应优先使用思源默认主题中真实存在的 `--b3-*` 变量；不确定是否存在的变量必须提供 fallback，不能假设所有主题都会提供同名变量。插件不能依赖私有、临时、非标准或只在某个主题中存在的变量作为核心样式基础。
