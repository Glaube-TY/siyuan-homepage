五、任务服务

新增 enhancedDiaryWorkspaceTaskService.ts。

必须复用 tasksPlusParser.ts：
- parseTaskLine
- generateTaskLine
- isTaskCompleted
- extractTaskTags

任务实体类型：

EnhancedDiaryWorkspaceTask {
  id: string;
  blockId: string;
  rootId?: string;
  box?: string;
  hpath?: string;
  markdown: string;
  taskname: string;
  completed: boolean;
  priority: string;
  startDate: string;
  deadline: string;
  recurrence: string;
  reminder: string;
  location: string;
  tags: string[];
  sourceKind: "new" | "migrated" | "normal";
  sourceDate?: string;
  sourceDocId?: string;
  sourceDocTitle?: string;
  isTodayTask: boolean;
  isOverdue: boolean;
  shouldMigrate: boolean;
}

实现函数：

1. queryWorkspaceTasks(config, today): Promise<EnhancedDiaryWorkspaceTask[]>
   - 使用现有 sql 查询任务块。
   - 优先参考 tasksPlus 组件现有 gettasksList / SQL 逻辑。
   - 不要重新发明和 Tasks Plus 冲突的查询。
   - 对每个任务用 parseTaskLine 解析。
   - 根据 root_id / hpath / 日记属性判断任务所在日记日期。
   - sourceKind：
     - 如果任务位于今天日记的 ## 新建任务 区块内 => new
     - 如果任务位于今天日记的 ## 迁移任务 区块内 => migrated
     - 其它 => normal
   - isTodayTask：
     - startDate 是今天，或 deadline 是今天，或任务在今天新建/迁移区块。
   - isOverdue：
     - 未完成，deadline 存在且 deadline < today。
   - shouldMigrate：
     - 未完成
     - sourceDate 存在
     - sourceDate 距 today 超过 config.taskMigrationReminderDays
     - 不在今天新建/迁移区块

2. toggleWorkspaceTaskComplete(task, completed)
   - 只更新任务块第一行任务状态。
   - 用 updateBlock("markdown", newMarkdown, task.blockId)。
   - 不覆盖整篇日记。
   - 保留任务下方子内容。

3. updateWorkspaceTask(task, input)
   - 用 generateTaskLine 重新生成第一行。
   - 保留原任务块第一行之后的子内容。
   - updateBlock 当前任务块。
   - 低风险处理：如果 getBlockKramdown 获取失败，不更新。

4. deleteWorkspaceTask(task, mode)
   - mode = "log" | "delete"
   - 如果 log：
     1. 找到今天日记。
     2. 找到 ## 任务动态。
     3. 写入：
        - 删除任务：任务名称（来源：((来源文档ID "来源文档标题"))）
     4. 写入日志成功后 deleteBlock 原任务块。
     5. 如果日志写入失败，不删除原任务。
   - 如果 delete：
     直接 deleteBlock，但必须由 UI 弹窗确认。

5. migrateWorkspaceTaskToToday(plugin, config, task)
   - 目标：任务只保留一个实体。
   - 找到或创建今日日记。
   - 找到 ## 迁移任务。
   - 优先使用 moveBlock 将原任务块移动到迁移任务区块末尾。
   - 迁移成功后，在任务块下方追加子内容：
     - 迁移来源：((来源日记文档ID "来源日记标题"))
     - 迁移时间：HH:mm
   - 如果 moveBlock 或定位失败，不删除原任务。
   - 如果无法可靠拿到移动后的任务块 ID，不要盲目追加到错误块；可以刷新任务列表后按 markdown 和目标日记定位。
   - 任何失败都必须返回清晰 reason，不要吞错误。

十一、API 和安全要求

1. 写入日记必须写到明确标题区块：
   - 新建任务 => # 今日日记 -> ## 新建任务
   - 迁移任务 => # 今日日记 -> ## 迁移任务
   - 快速记录 => # 今日日记 -> ## 快速记录 -> ### 分类
   - 删除日志 => # 今日日记 -> ## 任务动态
2. 找不到区块时提示补充模板，不写到文档末尾。
3. 任务迁移必须保证失败不删除原任务。
4. 删除任务必须弹窗确认。
5. 不要覆盖整篇日记。
6. 不要通过 exportMdContent 后 updateBlock 整个文档。
7. 更新任务只更新任务块。
8. 记录删除如果无法可靠定位多块范围，就先不支持硬删，不要乱删。
9. 所有操作完成后刷新 workspace state。
10. 所有错误要 console.warn，并给用户 showMessage 友好提示。
